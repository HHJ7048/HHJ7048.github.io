## APT 패키지 관리

### 리눅스 패키지

리눅스 소프트웨어를 실행하기 위한 모든 파일의 집합 (소프트웨어를 쉽고 안전하게 빠르게 설치가능)

1. 소스 패키지 : 사용자가 소스코드를 원하는 형태로 개조할 수 있는 형태
2. 바이너리 패키지 : 컴파일 완료된 바이너리를 제공, 설치 시간 단축, 오류 발생확률 낮아짐 다만 의존성 파일들을 제대로 제공하지 않는 경우 오류가 발생할 수 있다.
이러한 문제를 `패키지 의존성` 이라고 한다. 사용자가 모든 패키지 의존성을 고려하기는 어렵기 때문에 `패키지 관리자` 라는 도구를 활용한다.
    1. 저수준 패키지 관리 도구 : 단순히 자신의 패키지를 설치, 제거할 목적으로 사용된다.
    ex) dpkg, rpm 등이 있다.
    2. 고수준 패키지 관리 도구 : 패키지 간 의존성을 관리 도구가 스스로 파악하고 설치
    ex) apt, apt-get, yum, dnf 등이 있다.

### apt(advanced package tool) & apt-get

apt-get이 먼저 만들어졌으며 단점을 보완하기 위해 apt가 만들어졌다.
`apt-get + apt-cache + dpkg -l` 의 기능을 apt로 한번에 묶어서 처리했다.

#### 주로 사용하는 명령어와 옵션 ( apt [옵션] [명령어] )

| 명령어 | 기능 |
| --- | --- |
| list | 옵션을 만족하는 패키지 목록을 출력합니다 |
| search [검색어] | 검색어를 전체 패키지 목록에서 검색합니다 |
| show [패키지 이름] | 패키지 정보를 출력합니다 |
| install [패키지 이름] | 패키지를 시스템에 설치합니다 |
| reinstall [패키지 이름] | 패키지를 시스템에 재설치합니다 |
| remove [패키지 이름] | 패키지를 시스템에서 제거합니다 |
| update | 패키지를 다운로드하는 저장소의 정보를 업데이트합니다 |
| upgrade | 전체 패키지를 업그레이드 합니다 |

| 옵션 | 기능 |
| --- | --- |
| -d | 패키지를 시스템에 설치하지 않고 다운로드만 받습니다 |
| -y | 설치 시 사용자에게 설치 여부를 묻지 않고 항상 yes로 생각합니다 |
| --fix-missing | 패키지를 설치하거나 업데이트할 때 필요한 파일이 누락된 경우 재설치 및 복구를 진행합니다 |
| --upgradable | list 명령과 함께 사용 시 업그레이드 가능한 패키지 목록을 출력합니다 |
| --only-upgrade | 특정 패키지만 업그레이드 합니다 |
| --arch-only | 특정 아키텍처에 맞는 패키지에만 명령어를 적용합니다 |
| --dry-run | 패키지가 설치될 때의 변경점을 출력하며 실제로 설치하지는 않습니다 |
| --purge | 관련 설정파일까지 포함하여 특정 패키지를 완전히 제거합니다 |

## 네트워크 관리

컴퓨터들이 통신하기 위해 필요한 모든 것들을 말한다.
프로토콜 : 각 계층별로 사용하는 통신 규약( 사람으로 치면 언어와 같다.)

![](/assets/img/posts/Xpert%283%29/01.png)

기본적인 네트워크 지식은 넘어가겠다.

### 네트워크 인터페이스

컴퓨터나 네트워크 장비가 통신하기 위한 물리적 또는 가상의 연결 지점, `네트워크 인터페이스 카드(NIC)`는 LAN 혹은 WIFI로 전달 받은 전기 신호를 디지털 데이터로 변환하는 역할을 한다. 가상 머신에서는 `가상 랜 카드`가 네트워크 인터페이스 카드이다.

`$ ifconfig` : 네트워크 정보를 확인 할 수 있는 명령어

![](/assets/img/posts/Xpert%283%29/02.png)

| 옵션 | 기능 |
| --- | --- |
| flags | 인터페이스의 상태 |
| mtu | 인터페이스의 최대 전송 단위 |
| inet | 인터페이스에 할당된 IPv4주소 |
| netmask | 인터페이스 IPv4에서만 사용하는 기능<br>- ip주소와 netmask를 and연산하여 네트워크 주소를 구할 수 있음 |
| broadcast | 인터페이스의 브로드캐스트 주소 |
| ether | 인터페이스의 MAC주소 |
| RX[packets, bytes, ...] | 받은 패킷 정보 |
| TX[packets, bytes, ...] | 보낸 패킷 정보 |

`$ ifconfig [인터페이스 이름] up`  : 인터페이스 활성화
`$ ifconfig [인터페이스 이름] down`  :  인터페이스 비활성화

### 네트워크 접속 명령어

#### nc

netcat, TCP UDP 연결을 읽거나 기록하는 프로그램 / TCP UDP를 지원하는 네트워크 환경에 접속할 수 있다.

사용방법 : `nc [옵션] [ip] [port]` / `sudo apt install netcat` 을 설치해야 한다.

| 옵션 | 기능 |
| --- | --- |
| -d | 백그라운드 모드로 실행 |
| -e [프로그램 이름] | 네트워크 연결이 이루어진 경우 주어진 프로그램을 실행<br>이 옵션은 기본 `nc`에서는 비활성화 되어 있으며 `ncat` 등을 설치하여 사용 |
| -o [파일 경로] | 주어진 파일 경로에 통신 데이터를 저장 |
| -p [포트] | 로컬 포트를 지정 |
| -s [주소] | 로컬 주소를 지정 |
| -n | DNS를 사용하지 않음 |
| -u | UDP연결을 사용 |
| -l | 인바운드 연결을 지원하며 listen모드로 실행(수신자 역할을 수행) |
| -c | LF(`\n`) 대신 CRLF(`\r\n`)을 사용 |
| -v | 자세한 정보를 출력 |

#### ssh

secure shell,  `ssh`를 이용하여 사용자는 안전하게 원격으로 셸을 사용할 수 있다. 
디피-헬만 키 교환(Diffie-Hellman Key Exchange) 방식을 사용

port : 22번 사용

사용방법 : `ssh [옵션] [사용자 이름]@[서버 주소] [명령어]` / 명령어를 줄 경우 원격 시스템에 접속한 뒤 해당 명령을 실행한다.

| 옵션 | 기능 |
| --- | --- |
| -p [포트 번호] | 서버 주소의 포트번호로 접속<br>포트번호를 지정하지 않는 경우 22로 취급 |
| -v | 연결 과정에 대한 자세한 정보를 출력 |
| -i [private_key 경로] | 비밀번호 대신 private_key를 사용하여 접속 |
| -X | 접속 시 X11 애플리케이션을 실행할 수 있도록 함 |
| -L [로컬 포트]:[로컬 주소]:[원격 포트] | [로컬 주소]:[로컬 포트]를 원격 서버의 원격 포트로 포워딩<br>로컬은 ssh 클라이언트, 원격은 ssh 서버 |
| -R [로컬 포트]:[로컬 주소]:[원격 포트] | [로컬 주소]:[로컬 포트]를 원격 서버의 원격 포트로 포워딩<br>로컬은 ssh 서버, 원격은 ssh 클라이언트 |
| -C | 데이터를 압축하여 네트워크 성능을 향상. |

#### wget

web get, 웹 상의 파일을 다운로드 받기 위한 명령어 
HTTP, HTTPS, FTP 프로토콜을 사용하는 사이트의 파일들을 다운로드할 수 있다.

사용방법 : `wget [옵션] [URL]`

| 옵션 | 기능 |
| --- | --- |
| -O [파일명] | 다운로드한 파일의 파일명을 지정 |
| --limit-rate=N | 다운로드 시 속도를 제한 |
| -b | 백그라운드에서 다운로드 |
| -d | 디버그 내용을 출력 |
| --reject=[확장자] | 특정 확장자만 제외하고 다운로드 |
| -r | 재귀적으로 탐색하여 파일을 다운로드 |

`wget`을 이용해 파일을 가져올 수 있으며, `cat index.html`을 실행하면 다운로드 받은 파일의 내용을 확인할 수 있다.

#### curl

client URL, 특정 프로토콜을 이용해 URL로 데이터를 전송 / 서버에 데이터를 보내거나 가져올 때 사용
 HTTP, HTTPS, FTP, TELNET 등 다양한 프로토콜을 지원
사용방법 : `curl [옵션] [URL]`

| 옵션 | 기능 |
| --- | --- |
| -k | HTTPS를 사용하는 경우 인증서에 대한 검증 없이 연결 |
| -i | HTTP 헤더 정보만 출력 |
| -L | HTTP 301, 302를 받은 경우 리다이렉션 URL로 이동 |
| -d [데이터] | HTTP POST 데이터를 전송 |
| -o [파일 이름] | curl로 받아온 내용을 파일로 저장 |
| -X [메소드] | 주어진 메소드(GET, POST, DELETE 등)로 요청 |
| -A [이름] | User-Agent 정보를 전송 |
| -H [헤더] | 전송 시 사용할 헤더 정보를 지정 |
| -u [이름]:[암호] | 접속 시 이름과 비밀번호가 필요한 경우 사용 |

## 편집기(Editor)

cat, enter를 이용해서 문서 편집을 하면 불편한 경우가 많이 생긴다.

그래서 개발자들은 CLI환경을 위한 편집기를 사용하며 `vi, vim, emacs, nano` 등을 많이 사용

### vi, vim

#### vi

visual editor, 현재 가장 많이 사용되는 vim, neovim의 모태가 되는 만큼 기본적인 기능들을 포함하고 있다.

#### vim

visual editor improved 의 약자로 명령어모드, 편집모드 두가지 상태가 있다.

| 분류 | 명령어 | 기능 |
| --- | --- | --- |
| 편집 | i | 편집 모드로 진입, 커서 앞부터 작성 |
| 편집 | a | 편집 모드로 진입, 커서 뒤부터 작성 |
| 편집 | I | 편집 모드로 진입, 줄 맨 앞에서부터 작성 |
| 편집 | A | 편집 모드로 진입, 줄 맨 뒤에서부터 작성 |
| 이동 | w | 다음 단어 첫 글자로 이동 |
| 이동 | W | 공백 기준으로 다음 단어 첫 글자로 이동 |
| 이동 | b | 이전 단어 첫 글자로 이동 |
| 이동 | B | 공백 기준으로 이전 단어 첫 글자로 이동 |
| 이동 | gg | 문서 맨 위로 이동 |
| 이동 | G | 문서 맨 아래로 이동 |
| 이동 | ^ | 문장 맨 앞으로 이동 |
| 이동 | $ | 문장 맨 뒤로 이동 |
| 검색 | /[검색 내용] | 문서에서 [검색 내용]을 검색<br>- n과 N을 이용해 각각 다음 단어, 이전 단어로 이동 가능 |
| 편집 | y | 복사하기 |
| 편집 | c | 잘라내기 |
| 편집 | p | 붙여넣기 |
| 편집 | u | 실행 취소 |
| 편집 | ctrl + r | 재실행 |
| 편집 | dd | 현재 줄 잘라내기 |
| 편집 | yy | 현재 줄 복사하기 |
| 저장 | :w [파일 이름] | [파일 이름]으로 파일 저장하기<br>만약 `vim [파일 이름]`으로 미리 파일 이름을 지정했다면 `:w`만 사용 |
| 저장 | :q! | 저장하지 않고 편집기 종료 |
| 저장 | :wq | 저장 및 편집기 종료 |
