2017년 구글이 발표한 논문인 **"Attention is all you need"**에서 나온 모델로 기존 seq2seq의 인코더-디코더 구조를 따르면서, Attention만으로 구현한 모델이다. RNN을 사용하지 않았음에도 번역 성능이 RNN보다 뛰어나다.

> **아이디어** : 기존 seq2seq의 단점인 정보손실을 보정하기 위해 Attention이 사용되었다. 그러면 RNN을 보정하기 위한 용도가 아닌 Attention 만으로 Encoder와 Decoder를 설계해보면 어떨까?
{: .prompt-tip }

**트랜스포머의 주요 하이퍼파라미터** (논문에서 사용한 예시 값)

- $d_{model}$ : 인코더와 디코더에서 정해진 입출력의 크기 (512)
- $num\_layers$ : 인코더 디코더의 층 수 (6)
- $num\_heads$ : 병렬 분리 개수 (8)
- $d_{ff}$ : 은닉층의 크기 (2048)

## 1. Transformer 구조

인코더와 디코더가 여러 개 쌓여있는 구조를 가지고 있다.

![Transformer 구조](/assets/img/2026-07-16-LLM공부(6)/image(1).png)

RNN을 사용하지 않지만 여전히 인코더에서 입력받으면 디코더에서 출력해주는 구조를 가지고 있다.

## 2. Positional Encoding

RNN은 단어를 순차적으로 입력받아 포지션 정보를 가질 수 있다는 장점이 있다. 하지만 RNN을 사용하지 않는 트랜스포머는 포지션 정보를 반영할 수 없다. 그래서 위치 정보를 다른 방식으로 알려줘야 한다.

트랜스포머에서는 임베딩 벡터들이 입력으로 사용되기 전에 Positional Encoding 값이랑 더해지는 방식으로 위치 정보를 반영한다.

![Positional Encoding 더하기](/assets/img/2026-07-16-LLM공부(6)/image(2).png)

여기서 Positional Encoding의 값은 다음과 같은 식을 따른다.

![Positional Encoding 수식](/assets/img/2026-07-16-LLM공부(6)/image(3).png)

여기서 $pos$는 문장 내 단어의 순서이고 $i$는 임베딩 벡터 내의 차원 인덱스를 의미한다.

$pos$ 값을 그대로 사용하면 문장 길이가 길어질 경우 값이 커진다.

**sin 함수와 cos 함수를 사용하면**

- 값을 $[-1, 1]$로 조절해 인코딩 값이 폭발하는 문제를 해결할 수 있다.
- $1/10000^{2i/d_{model}}$ 이 부분이 주파수로, $i$마다 다른 주파수를 만들어 각 위치마다 고유한 패턴을 생성해 구분할 수 있다.
- 기존 차원이 정해진 PE를 만들었다면 sin cos 사용으로 동적인 크기의 PE를 만들 수 있다.
- sin cos 쌍을 사용해서 임의의 위치 $pos + k$의 인코딩을 $pos$ 인코딩의 선형 변환으로 나타내서 상대적 위치를 학습하기 쉽다.

참고 강의 자료 : <https://youtu.be/I1OMC3A15HU?si=tHhpW3vGPC3p_skX>

50 x 128의 크기를 가진 포지셔널 인코딩 행렬을 시각화하면 아래와 같은 형태가 나온다.

![Positional Encoding 시각화](/assets/img/2026-07-16-LLM공부(6)/image(4).png)

- Depth가 낮을수록 고주파를 형성한다. 따라서 잘게 요동치는 형태의 모양이 나온다.
- 반대로 높아질수록 초저주파를 형성하게 되어서 색이 잘 변하지 않는다.
- sin인 경우 짝수 차원으로 흰색이고 cos인 경우 홀수 차원으로 파란색을 띈다. 이를 통해 sin cos을 쌍으로 배치했다는 것을 확인할 수 있다.

### +RoPE(Rotary Position Embedding)

- 기존의 PE는 위치 패턴을 단순히 더한다.
- 의미정보에 영향을 줄 수 있다.
- 단어의 관계에 있어서 상대위치가 중요하다.

> **아이디어** : 단어를 더하지 말고 회전시키자 (더 알아보기)

## 3. Encoder, Attention의 종류

### 1. Attention의 종류

트랜스포머에서 사용되는 세 가지 Attention의 종류 정리

| Attention 종류 | 사용 위치 | Q, K, V 출처 |
| --- | --- | --- |
| Encoder Self-Attention | 인코더 | Query = Key = Value |
| Masked Decoder Self-Attention | 디코더 | Query = Key = Value |
| Encoder-Decoder Attention | 디코더 (인코더의 값을 불러옴) | Key = Value |

(여기서 같다라는 의미는 벡터의 값이 아니라 **출처가 같다**는 의미로 이해)

Multi-head란 트랜스포머가 어텐션을 병렬적으로 수행하는 방법을 의미한다.

### 2. Self-Attention

자기 자신에게 수행하는 Attention이다. 자신의 문장에 있는 Q로 자신의 문장과 Attention시킨다.

Q, K, V : 입력 문장의 모든 단어 벡터들
- $d_{model}$의 차원보다 더 작은 차원을 가지게 된다. (논문에서는 64차원을 가진다.)

![Q, K, V 벡터 변환 과정](/assets/img/2026-07-16-LLM공부(6)/image(5).png)
![Q, K, V 벡터 변환 과정](/assets/img/2026-07-16-LLM공부(6)/image(6).png)

student라는 단어를 Q, K, V 벡터로 변환하는 과정이다.

![Scaled dot-product Attention](/assets/img/2026-07-16-LLM공부(6)/image(7).png)


### 3. Scaled dot-product Attention

앞서 바다나우 어텐션에서 설명한 대로 차원이 커질 경우 내적 값이 폭발하는 문제를 해결한 dot-product Attention 방법론이다.

$$\sqrt{d_k}, \quad d_k = \frac{d_{model}}{num\_heads}$$

$\sqrt{d_k}$를 나눠줌으로써 값을 스케일링 해준다.

![행렬연산 도식화](/assets/img/2026-07-16-LLM공부(6)/image(8).png)

실제로는 Q와 K의 내적은 행렬연산으로 진행된다. 그래서 전체 식을 도식화하면 이렇게 된다. → GPU가 행렬연산을 아주 빠르게 수행할 수 있다.

![행렬연산 도식화](/assets/img/2026-07-16-LLM공부(6)/image(9).png)
![행렬연산 도식화](/assets/img/2026-07-16-LLM공부(6)/image(10).png)

### 4. Multi-head Attention

Attention을 병렬로 수행하면 여러 시각에서 들어오는 정보를 다 수집할 수 있다. (Attention은 각기 다른 시각에서 유사도를 측정하기 때문이다.)

여기서 쪼갠다는 것은 key의 갯수가 아니고 **각 key의 차원**이다. 차원을 단순히 버리는 게 아니라 가중치를 곱해서 수학적으로 차원을 축소시킨다. 따라서 하나의 벡터는 의미가 희석될 수 있지만 여러 개를 만들어 합치면 서로 상호보완된다.

![행렬연산 도식화](/assets/img/2026-07-16-LLM공부(6)/image(11).png)
![행렬연산 도식화](/assets/img/2026-07-16-LLM공부(6)/image(12).png)

1. 초기 가중치들은 랜덤으로 정해져 차원을 축소한다.
2. 축소된 a들을 concat해서 이어붙힌다.
3. 이어붙힌 행렬을 $W_0$를 행렬곱해서 크기는 그대로지만 모든 의미를 재조합해준다. (다 따로 노는 것을 하나로 합쳐줌)

![행렬연산 도식화](/assets/img/2026-07-16-LLM공부(6)/image(13).png)

위의 그림의 차원과 별개로 3번 단계를 설명하기 위한 그림

### 5. Padding Mask

일반적으로 max_len은 batch마다 seq_len의 길이 중 가장 긴 문장의 길이를 기준으로 한다.
- 95%~99%의 문장을 커버하는 선에서 고정값을 사용하는 경우도 있다.

Padding Mask 처리는 Attention Score를 구하고 나서 Padding Mask(패딩 처리를 위한 벡터)를 더해서 패딩 처리를 한다.

![Padding Mask](/assets/img/2026-07-16-LLM공부(6)/image(14).png)

그리고 Softmax 함수를 돌리면 이 부분은 0이 되어 무의미한 값을 가지게 된다.

> pad를 하는 이유는 batch로 묶어서 한 번에 병렬연산 하기 위해서 길이를 맞춰주는 거다.
{: .prompt-info }

### 6. Position-Wise FFNN(완전 연결 FFNN)

$x$는 $(seq\_len, d_{model})$의 크기를 가지고 $W_1$은 $(d_{model}, d_{ff})$의 크기를 가진다. $W_2$는 $(d_{ff}, d_{model})$의 크기를 가진다.

$$FFNN(x) = MAX(0,\ xW_1 + b_1)W_2 + b_2$$

![Position-Wise FFNN 구조](/assets/img/2026-07-16-LLM공부(6)/image(15).png)

여기서 매개변수 $W_1, b_1, W_2, b_2$는 하나의 인코더 층에서 다른 문장, 단어들마다 동일하게 사용된다.
- 인코더 층마다는 다른 값을 가진다.

**하나의 층에서 같은 가중치를 쓰는 이유**
1. 일반화가 가능
2. 파라미터 갯수의 효율성

![ADD & NORM](/assets/img/2026-07-16-LLM공부(6)/image(16).png)

**Position-Wise FFNN을 사용하는 이유**
1. 비선형적인 층을 추가해서 더 복잡한 연산을 가능하게 해주기 때문이다.
2. 입출력 차원을 $d_{model}$로 유지해서 계속 층을 추가할 수 있다.

### 7. 잔차 연결(Residual connection), 층 정규화(Layer Normalization)

인코더에서 ADD & NORM에 해당하는 부분

![ADD & NORM](/assets/img/2026-07-16-LLM공부(6)/image(17).png)

#### 1. Residual connection

Sub Layer Input + Output을 진행한다. 이는 컴퓨터 비전에서 주로 사용되는 모델 학습 기법이다.

![ADD & NORM](/assets/img/2026-07-16-LLM공부(6)/image(18).png)

1. 어텐션을 사용한 $F(x)$를 바로 사용하는 것보다 $x$에서 $F(x)$만큼을 조금씩 학습해 나가는 것이 더 쉽고 안전하게 학습할 수 있다.
2. 기울기가 잘 흐른다. 미분하면 $1 + F'(x)$
3. 정보손실을 방지한다.

관련 논문 : <https://arxiv.org/pdf/1512.03385.pdf>

**단점** : 계속 더해지면 기울기가 폭발할 수 있다.

#### 2. Layer Normalization

**내부 공변량 변화** : 층을 쌓다 보면 각 층마다의 크기 차이가 심한 경우가 있다. 이럴 경우 기울기가 불안정해져 학습 과정이 불안정해진다.

층 정규화는 이러한 문제를 해결하기 위해 순전파 과정에서 값을 조절해 완충제 역할을 해준다.

$ln_i = LayerNorm(x_i)$

**1. 평균 분산을 이용**

텐서의 마지막 차원에서 평균과 분산을 구하고 아래 수식을 통해 값을 정규화하여 학습을 돕는다.

![평균 분산 정규화 수식 1](/assets/img/2026-07-16-LLM공부(6)/image(19).png)

![평균 분산 정규화 수식 2](/assets/img/2026-07-16-LLM공부(6)/image(20).png)

- 입실론은 분모 0 방지
- 핵심은 분자와 분모가 같은 표본에서 나온 값이기 때문에 결국 스케일이 줄어듦 (모든 벡터가 평균 0, 분산 1로 만들어짐)

**2. 감마 베타를 이용**

평균 분산을 이용한 정규화에서 값을 다시 유연성 있게 만들어주는 방법이다.

감마 베타의 초기값이다.

![감마 베타 초기값](/assets/img/2026-07-16-LLM공부(6)/image(21).png)

감마는 스케일을 조절, 베타는 값을 이동시키는 목적

![감마 베타 수식](/assets/img/2026-07-16-LLM공부(6)/image(22).png)

감마 베타 역시 역전파 과정에서 학습해서 값을 조절한다.

## 4. Decoder

### 1. Masked Multi-head Self-Attention

디코더는 쿼리를 보고 단어를 예측해야 하지만 어텐션이 전체 문장을 파악하기 때문에 문제가 발생할 수 있음. 이를 방지하기 위해서 **Look-ahead mask**를 도입했다. / 물론 teacher forcing 학습 과정에서 사용된다. 추론에서는 `<pad>`를 처리하는 코드만 동작한다.

![Look-ahead mask](/assets/img/2026-07-16-LLM공부(6)/image(23).png)

첫 번째 서브층에서 사용된다. Masked가 붙은 이유이다.

기존 Self-Attention에서 padding은 Look-ahead mask에 포함되어서 같이 진행된다.

### 2. Encoder-Decoder Attention

K, V의 출처가 인코더이기 때문에 Self-Attention이 아니다. 인코더의 Key, Value를 이용해서 디코더의 Query와 유사도를 어텐션한다.

<hr/>

> **Q. 디코더 두 번째 서브층에서 인코더의 출력을 가져오게 되면 결국에는 미래 문장을 반영하게 되는 것이 아닐까?**
>
> A. 인코더에서 받아오는 것은 원문 문장이고 디코더에서 막아야 할 문장은 생성 중인 미래 문장이기 때문에 상관없다.
{: .prompt-tip }

> **Q. 디코더와 인코더를 하나만 사용할 순 없을까?**
>
> A. look-ahead mask로 인해 학습 시 뒤의 문맥을 파악하기가 힘들다. 그래서 인코더를 사용한다. GPT는 뒤 문맥이라는 것 자체가 존재하지 않는 작업이기 때문에 (그 순간 순간 만들어낸다.) 디코더만 사용해서 구성할 수 있다.
{: .prompt-tip }
