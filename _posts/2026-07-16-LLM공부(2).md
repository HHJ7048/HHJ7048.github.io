Integer Encoding은 정수의 크기가 잘못된 정보를 전달 할 수 있다는 문제가 있었다. 이를 방지하기 위해 단순 정수가 아닌 Vectorization하는 과정이 필요하다. One-hot Encoding과 DTM, TF-IDF에 대해서 정리하겠다.

## 1. One-hot Encoding

대표적인 Vectorization 방식이다.

- 1 -> 1 0 0 0
- 2 -> 0 1 0 0
- 3 -> 0 0 1 0

와 같이 정수를 벡터화 시킨다.

![이미지](/assets/img/2026-07-16-LLM공부(2)/image(10).png){: width="400" }

### 단점

무의미한 0과 1로는 데이터들의 관계를 설명할 수 없다.
따라서 벡터화된 데이터를 Word Embedding이라는 방식으로 실수화 시켜서 데이터들의 관계를 표현한다.

## 2. DTM과 TF-IDF

### 1. DTM

Document-Term Matrix의 약자로 문서에서 단어의 빈도수를 행렬로 나타낸 것이다.

![이미지](/assets/img/2026-07-16-LLM공부(2)/image.png){: width="700" }

tf(d, t) 는 문서에서의 단어의 개수를 의미한다.

#### 한계

Vocabulary size가 커지면 0의 개수가 너무 많아져 데이터의 크기가 과하게 커질 수 있다. 이로인해 행렬의 희소 문제가 발생한다.

### 2. TF-IDF

Term Frequency * Inverse Document Frequency의 약자이다.

해당 문서의 단어 빈도 * 해당 단어가 나온 문서의 빈도 역수를 계산한 값을 가중치로 곱해준 table이다.

여기서 idf(t)는 t가 나온 문서의 빈도를 뜻한다. 식은 다음과 같다.

![이미지](/assets/img/2026-07-16-LLM공부(2)/image(1).png){: width="300" }

n이 너무 커지면 idf가 너무 커져 과하게 영향을 줄 수 있기 때문에 log를 이용해 값을 눌러준다.

df(t)가 0이 될 경우 분모가 0이 되어서 오류가 발생하는 것을 방지하기 위해 1을 더해준다.

ㄴdf(t)가 0이라는 것은 한번도 나오지 않은 단어 OOV(Out-Of-Vocabulary)에 대한 예측을 의미한다.

![이미지](/assets/img/2026-07-16-LLM공부(2)/image(2).png){: width="700" }

DTM 에 TF-IDF가중치를 곱한다.

단순히 특정 문서에서 TF가 많이 사용되었을 때 그 문서에서만 자주 사용되는 단어인지 전체적으로 자주 사용되는 단어인지 구분해줌

- 그 문서에서만 자주 사용 → 실질적으로 예측해야 하는 단어일 확률이 높음
- 모든 문서에서 자주 사용 → 은 는 이 가 와 같이 정보가치가 낮은 단어일 확률이 높음
