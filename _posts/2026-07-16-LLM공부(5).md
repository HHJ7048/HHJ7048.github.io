## 1. Machine Translation History

SMT(통계 기반 기계 번역)에서 NMT(신경망 기계 번역) 으로 발전되었다.

### Neural Machine Translation의 장점

**1. Word Embedding으로 인한 Continuous Representation**

Training Data에 없던 단어도 예측이 가능하고 번역할 수 있게 됐다.

**2. End-to-End 모델로의 전환**

기존 SMT는 여러 개의 독립된 모듈(정렬 모델, 언어 모델, 번역 모델 등)로 이루어져 있어서 모듈간 오차가 발생하는 문제가 생겼지만 신경망 번역은 End-to-End모델로 입력부터 출력까지 하나의 모델에서 진행하기 때문에 이러한 문제를 해결했다.

**3. Attention 매커니즘의 긴 문장 처리 성능 개선**

초기 인코더는 하나에 입력 문장 전체 정보를 압축해야 하는 번거로움이 있었다. Attention이 도입되면서 디코더가 매 시점마다 인코더의 모든 hidden state에 접근할 수 있게 되었고 긴 문장도 정확하게 번역할 수 있게 되었다.

## 2. Sequence to Sequence

LSTM 모델이 각각 인코더와 디코더로 사용된 신경망 기반 기계 번역 모델이다.

### 동작 원리

![이미지](/assets/img/2026-07-16-LLM공부(5)/image(1).png){: width="700" }

사용자의 입력이 Encoder에서 Embedding되어 LSTM에 입력된다.

입력이 끝난 후 마지막 단어가 들어간 LSTM의 hidden state(CONTEXT로 표기)가 첫 번째 디코더 LSTM의 hidden state로 입력된다. 여기서 CONTEXT는 입력들의 정보를 압축한 하나의 벡터이다.

먼저 &lt;sos&gt;가 입력되면 생성이 시작된다. &lt;eos&gt;가 입력될 때 까지 단어를 생성한다.

### Sequence2Sequence의 Teacher forcing 학습

Decoder에서 입력에 대한 출력을 미리 정해놓고 예측 과정 없이 빠르게 학습시키는 방법

![이미지](/assets/img/2026-07-16-LLM공부(5)/image(2).png){: width="250" }
![이미지](/assets/img/2026-07-16-LLM공부(5)/image(3).png){: width="250" }

실제 학습에서는 Overfitting을 방지하기 위해 일부 데이터는 Teacher forcing을 사용하지 않고 학습시킨다.

### Sequence to Sequence 단점

**1. Bottleneck 현상**

입력 단어의 길이가 길어질수록 하나의 CONTEXT vector에 압축해서 담을 경우 정보 손실이 일어날 수 있다.

CONTEXT Vector는 일반적으로 512차원의 크기를 가진다.

**2. long-term dependency**

LSTM에서 hidden state는 정보를 계속해서 삭제하는 특성을 가지고 있다.

길이가 길어지면 그만큼 삭제하는 정보량이 점점 많아지게 되고 정보가 장기적으로 유지되지 못할 수 도 있다.

## 4. Dot-Product Attention

Sequence2Sequence에서 사용되는 Attention Mechanism중 하나의 종류이다.

기존 문제점인 Bottleneck과 Long-term dependency를 해결하기 위해 나온 방법론이다.

### 동작원리

1. Query에 대한 각 Key를 Dot-Product해서 유사도를 구한다. Attention Score이라고 한다.

2. Softmax함수를 이용해 유사도의 합을 1로 만든다. 이렇게 만들어진 유사도의 분포를 Attention Distribution이라고 한다. 또한 각각의 유사도 값을 Attention Weight이라고 한다.

3. 각 key의 Value를 곱한다. 그리고 모두 합친다. 이렇게 구해진 값을 Attention Value라고 하고 수식에서는 at로 입력한다.

4. at를 기존 Query와 concatenate한다.

    ![이미지](/assets/img/2026-07-16-LLM공부(5)/image(5).png){: width="450" }

5. 가중치 Wc와 곱하고 하이퍼탄젠트볼릭 함수를 사용해서 새로운 hidden state를 만든다.

    ![이미지](/assets/img/2026-07-16-LLM공부(5)/image(6).png){: width="450" }

6. 출력부분에서 새로 만들어진 hidden state를 사용한다.

    ![이미지](/assets/img/2026-07-16-LLM공부(5)/image(14).png){: width="400" }

### 단점

긴 글에서 정보 손실을 줄여 정확하게 사용가능하지만 1번 과정에서 Dot-Product하는 과정에서 데이터의 길이가 길어질 경우 연산량이 많아질 수 있다.

## Attention 매커니즘의 종류

Dot-product Attention 보다 조금 더 복잡하게 설계된 Bahdanau Attention의 개념과 Luong Attention(Dot-product Attention)을 비교해보겠다.

![이미지](/assets/img/2026-07-16-LLM공부(5)/image(7).png){: width="800" }

### 1. Bahdanau Attention

#### 1. Bahdanau Attention 함수

Attention(Q, K, V) = Attention Value

| 기호 | 의미 |
| --- | --- |
| t | 어텐션 메커니즘이 수행되는 디코더 셀의 현재 시점을 의미 |
| Q | t-1 시점의 디코더 셀에서의 은닉 상태 |
| K | 모든 시점의 인코더 셀의 은닉 상태들 |
| V | 모든 시점의 인코더 셀의 은닉 상태들 |

#### 2. Bahdanau Attention 동작원리

**1. 어텐션 스코어를 구한다**

앞서 설명한 Dot-Product Attention과 달리 단순 내적이 아닌 하이퍼볼릭 탄젠트 함수를 이용한다.

![이미지](/assets/img/2026-07-16-LLM공부(5)/image(12).png){: width="350" }

tanh() 안에 있는 Wb·s(t-1) 와 Wc·h(i) 를 먼저 각각 구해보면

![이미지](/assets/img/2026-07-16-LLM공부(5)/image(8).png){: width="450" }

![이미지](/assets/img/2026-07-16-LLM공부(5)/image(9).png){: width="450" }

에서 WaT를 곱해서 유사도가 기록된 어텐션 벡터 et를 구한다.

![이미지](/assets/img/2026-07-16-LLM공부(5)/image(10).png){: width="450" }

여기서 WaT를 곱하는 이유는 tanh(~)의 차원인 4x4를 4x1로 만들어 주기 위함이다. 왜냐하면 Attention에 대한 score는 각 key당 하나의 값을 가져야하기 때문이다.

**2. 컨텍스트 벡터로 st를 구한다.**

기존의 LSTM이 임베딩된 단어 벡터를 입력으로 하는 것에서 컨텍스트 벡터와 임베딩된 단어 벡터를 concatenate하여 입력하는 것으로 달라졌다.

![이미지](/assets/img/2026-07-16-LLM공부(5)/image(13).png){: width="500" }

**3. 나머지 과정은 Dot-product Attention과 동일하다.**

### 2. Bahdanau Attention의 장단점

**장점**

- 차원의 제약이 없어짐
- 학습되는 유사도로 표현력이 증가됨
- 비선형 유사도 함수로 학습함 (내적이라는 고정된 가정에서 벗어남)

**단점**

- 벡터별 tanh 사용으로 느려짐
- 파라미터가 늘어서 과적합 위험 증가

> **\+ Dot-Product 발표간 질문 : 트레이드오프 문제를 해결할 수 있는 방안이 있을까? 와 관련된 내용**
>
> 루옹(dot-product)논문에서 실험해본 결과 성능 차이가 크지않았고 결국 Transformer는 Bahdanau가 아닌 Dot-Product를 선택했다.
>
> - Q, K를 만들 때 WQ WK를 곱하니깐 학습 가능한 파라미터와 차원 맞춤은 해결됨
> - 스코어 함수 자체가 복잡할 필요 없이 빠른 내적 계산이 최선(GPU 사용 시 매우 빨라짐, 병렬화가 쉬워짐)
> - 대신 차원이 커질경우 발생하는 내적값 폭발 문제만 루트dk로 나눠서 보정 시킴 -> scaled dot-product
