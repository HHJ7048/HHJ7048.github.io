# RNN

Recurrent Neural Network로 입력과 출력을 시퀀스 단위로 처리하는 시퀀스 모델이다.
주요 특징은 hidden state를 재귀적으로 전달해서 이전까지의 입력 상태를 현재 출력에 반영한다. 덕분에 앞(뒤) 문맥을 파악할 수 있다. 또한 같은 가중치를 공유해서 가변 길이의 입력을 처리할 수 있다.

RNN의 종류로는 LSTM과 GRU가 있고 LSTM은 seq2seq 모델을 이해하는데 가장 중요한 부분이다.

# RNN(Recurrent Neural Network)의 개념

![이미지](/assets/img/2026-07-16-LLM공부(9)/image.png){: width="500" }
![이미지](/assets/img/2026-07-16-LLM공부(9)/image%201.png){: width="200" }

피드 포워드 신경망은 은닉층의 값이 출력층 방향으로만 흐른다. 반면 RNN은 은닉층 노드의 결과를 출력층으로 보내는 동시에, 다음 시점 자기 자신의 입력으로도 보낸다.

- **셀(cell)** : 은닉층에서 활성화 함수를 거쳐 결과를 내보내는 노드. 이전 값을 기억하는 메모리 역할을 하므로 메모리 셀 또는 RNN 셀이라고 부른다.
- **은닉 상태(hidden state)** : 메모리 셀이 출력층과 다음 시점 t+1의 자신에게 보내는 값. t 시점의 셀은 t-1 시점의 은닉 상태를 입력으로 받는다.

RNN은 입력과 출력의 길이를 용도에 맞게 다르게 설정 가능하다.

![이미지](/assets/img/2026-07-16-LLM공부(9)/image%202.png){: width="600" }

- 일 대 다(one-to-many) : 이미지 캡셔닝
- 다 대 일(many-to-one) : 감성 분류, 스팸 메일 분류
- 다 대 다(many-to-many) : 번역기, 챗봇, 개체명 인식, 품사 태깅

![이미지](/assets/img/2026-07-16-LLM공부(9)/image%203.png){: width="600" }

$h_t = \tanh(W_x x_t + W_h h_{t-1} + b)$

$y_t = f(W_y h_t + b)$

$f$는 비선형 활성화 함수이며, 이진 분류면 시그모이드, 다중 클래스 분류면 소프트맥스를 쓴다. 은닉 상태 계산에는 주로 tanh를 사용한다.

$W_x, W_h, W_y$는 **한 층 안에서는 모든 시점이 값을 동일하게 공유한다.** 덕분에 시퀀스 길이가 변해도 파라미터 수는 그대로다. 다만 은닉층이 2개 이상이면 층마다 가중치는 서로 다르다.

# RNN의 종류

## Deep RNN

![이미지](/assets/img/2026-07-16-LLM공부(9)/image%204.png){: width="600" }

은닉층을 2개 이상 쌓은 구조다. 첫 번째 은닉층은 위에 다음 은닉층이 존재하므로, 마지막 시점만이 아니라 모든 시점의 은닉 상태를 위층으로 넘겨줘야 한다.

은닉층을 추가하면 학습할 수 있는 양은 늘어나지만 그만큼 훈련 데이터도 많이 필요하다.

## BiRNN

![이미지](/assets/img/2026-07-16-LLM공부(9)/image%205.png){: width="600" }

아이디어 : 시점 t의 출력값을 예측할 때 이전 시점의 입력뿐 아니라 이후 시점의 입력도 예측에 기여할 수 있다.

그래서 양방향 RNN은 하나의 출력값을 예측하는 데 **두 개의 메모리 셀**을 사용한다.

- **Forward States** : 앞 시점의 은닉 상태를 전달받아 현재 은닉 상태를 계산
- **Backward States** : 뒤 시점의 은닉 상태를 전달받아 계산

이 두 값 모두가 현재 시점 출력층의 입력으로 쓰인다.

## LSTM

문장이 길어질수록 앞쪽 입력의 정보량이 뒤로 충분히 전달되지 못하고 점점 희석된다. 이를 **장기 의존성 문제(the problem of Long-Term Dependencies)** 라고 한다.

구조적 원인은 은닉 상태가 매 시점 $W_h$와 tanh를 반복해서 통과한다는 데 있다. 역전파(BPTT) 과정에서 같은 가중치가 시점 수만큼 곱해지므로 기울기가 지수적으로 줄어들거나(소실) 커진다(폭주).

![이미지](/assets/img/2026-07-16-LLM공부(9)/image%206.png){: width="600" }

정보를 매 시점 통째로 덮어쓰는 대신, **반복 행렬곱을 거치지 않고 흘러가는 별도의 통로**를 하나 더 만든다. 이것이 **셀 상태(cell state)** 라고 한다. 

- 셀 상태는 $C_t = f_t \circ C_{t-1} + i_t \circ g_t$ 처럼 **덧셈**으로 갱신된다. 이전 값이 $W_h$에 반복해서 곱해지지 않으므로 기울기가 먼 시점까지 살아남는다.

은닉 상태를 계산하는 식이 복잡해진 대신, 셀 상태라는 장기 기억 통로가 추가된 RNN이다. 셀 상태를 장기 상태, 은닉 상태를 단기 상태라고 부르기도 한다.

### 입력 게이트

![이미지](/assets/img/2026-07-16-LLM공부(9)/image%207.png){: width="600" }

$i_t = \sigma(W_{xi}x_t + W_{hi}h_{t-1} + b_i)$

$g_t = \tanh(W_{xg}x_t + W_{hg}h_{t-1} + b_g)$

현재 정보를 기억하기 위한 게이트다. 현재 시점 t의 $x_t$와 이전 시점의 $h_{t-1}$을 각각 가중치와 곱해 더한 뒤, 두 가지 활성화 함수로 나누어 보낸다.

- $i_t$ : 시그모이드를 지나 **0~1** 값. 이번에 들어온 정보를 얼마나 반영할지에 해당하는 ‘양’.
- $g_t$ : tanh를 지나 **-1~1** 값. 이번에 기억할 후보 정보 자체에 해당하는 ‘내용’.

### 삭제 게이트

![이미지](/assets/img/2026-07-16-LLM공부(9)/image%208.png){: width="600" }

$f_t = \sigma(W_{xf}x_t + W_{hf}h_{t-1} + b_f)$

기억을 삭제하기 위한 게이트다. 현재 시점 t의 $x_t$와 이전 시점의 $h_{t-1}$이 시그모이드 함수를 지나 0~1 사이의 값이 되고, 이 값이 곧 삭제 과정을 거친 정보의 양이다.

- 0에 가까울수록 이전 기억을 많이 삭제한 것
- 1에 가까울수록 이전 기억을 온전히 유지한 것

### 셀 상태

![이미지](/assets/img/2026-07-16-LLM공부(9)/image%209.png){: width="600" }

$C_t = f_t \circ C_{t-1} + i_t \circ g_t$

삭제 게이트에서 일부 기억을 잃은 이전 셀 상태 $f_t \circ C_{t-1}$에, 입력 게이트에서 구한 $i_t \circ g_t$를 더한다. 이 값이 현재 시점 t의 셀 상태이며 다음 t+1 시점의 LSTM 셀로 넘겨진다. $\circ$ 는 같은 크기의 두 행렬에서 같은 위치의 성분끼리 곱하는 원소별 곱이다.

- $f_t = 0$ 이면 $C_{t-1}$의 영향력이 0이 되어 **입력 게이트의 결과만**이 $C_t$를 결정한다. (삭제 게이트를 완전히 닫고 입력 게이트를 열어둔 상태)
- $i_t = 0$ 이면 $C_t$는 오직 $C_{t-1}$에만 의존한다. (입력 게이트를 닫고 삭제 게이트만 열어둔 상태)

### 출력 게이트

![이미지](/assets/img/2026-07-16-LLM공부(9)/image%2010.png){: width="600" }

$o_t = \sigma(W_{xo}x_t + W_{ho}h_{t-1} + b_o)$

$h_t = o_t \circ \tanh(C_t)$

출력 게이트 $o_t$는 현재 시점 t의 $x_t$와 이전 시점의 $h_{t-1}$이 시그모이드 함수를 지난 값이고, 현재 시점의 은닉 상태를 결정하는 데 쓰인다.

셀 상태 $C_t$가 tanh를 지나 -1~1 사이 값이 되고, 여기에 출력 게이트 값이 원소별로 곱해지면서 값이 걸러지는 효과가 발생해 은닉 상태가 된다. 즉 **장기 기억(셀 상태) 중 이번 시점에 내보낼 부분만 골라내는 역할**이다. 은닉 상태는 다음 시점으로 전달되는 동시에 출력층으로도 향한다.

## GRU

![이미지](/assets/img/2026-07-16-LLM공부(9)/image%2011.png){: width="600" }

LSTM의 3개 게이트(입력·삭제·출력)를 2개(업데이트·리셋)로 줄이고, 셀 상태 없이 은닉 상태 하나만 사용한다.

**리셋 게이트**

$r_t = \sigma(W_{xr}x_t + W_{hr}h_{t-1} + b_r)$

**업데이트 게이트**

$z_t = \sigma(W_{xz}x_t + W_{hz}h_{t-1} + b_z)$

**후보 은닉 상태**

$g_t = \tanh(W_{hg}(r_t \circ h_{t-1}) + W_{xg}x_t + b_g)$

**은닉 상태**

$h_t = (1 - z_t) \circ g_t + z_t \circ h_{t-1}$

LSTM보다 단순한 구조로 만들었지만 성능이 더 좋아졌다는 결과는 아직 없다.
