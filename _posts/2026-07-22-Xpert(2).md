## 가상머신(Virtual Machine, VM)

하나의 컴퓨터 안에서 또 하나의 컴퓨터를 에뮬레이션 하는 소프트웨어.

1. 하나의 운영체제 위에 다양한 운영체제 환경을 구축할 수 있다.
2. 독립된 환경을 조성해서 악성 코드 분석 등과 같이 위험한 작업을 할 때 호스트 컴퓨터에 피해가 가지 않도록 한다.
3. 스냅샷 기능을 활용할 수 있다. 이 기능은 가상 머신의 상태를 저장하고 필요할 때 빠르게 복원할 수 있게 해준다.

## 리눅스 시스템

- **Shell** : 유저가 리눅스 시스템을 이용할 수 있는 인터페이스 (`Ctrl` + `Alt` + `T`)
- **Shell Prompt** : `user@user-VirtualBox:~$` 이 부분으로 shell 명령어를 입력받을 준비가 되었음을 알 수 있다.

### 와일드카드

리눅스에서 임의의 다른 문자를 나타낼 수 있는 특수문자.
여러 파일을 한번에 관리하기 위해서 사용한다. (예: `rm *.txt` → 모든 txt 파일 삭제)

| 와일드카드 | 설명 |
| --- | --- |
| `?` | 임의의 1개 문자로 대체 (문자 종류 제한 없음) |
| `*` | 임의의 0개 이상의 문자로 대체 |
| `[...]` | 괄호 안의 문자 중 하나 |
| `{a,b}` | 나열한 것들로 각각 확장 |

### 리다이렉션

명령어의 입출력 방향을 바꾸는 것. `>` `>>` `<`

```bash
ls > list.txt      # 출력을 파일에 저장 (덮어쓰기)
ls >> list.txt     # 파일 끝에 이어붙이기
wc -l < list.txt   # 파일을 입력으로 받기
```

### 파이프

명령어의 출력을 다른 명령어의 입력으로 연결한다. `|`

```bash
ls -l | grep .py
```

## 권한

### User와 Group

유저는 고유한 사용자 ID를 가지고 있다. 그룹은 유저가 속할 수 있는 집합이다.

```bash
cat /etc/passwd   # 리눅스의 유저 정보를 담고 있는 파일을 출력
cat /etc/group    # 리눅스의 그룹 정보를 담고 있는 파일을 출력
```

### 파일 및 디렉토리 권한

`ls -l` 을 사용했을 때 나오는 결과의 예시이다.

```text
drwxrwxr-x 2 user user 4096 12월  2 13:08 dir
```

권한 플래그 : `d` `rwx` `rwx` `r-x`

1. `d`는 디렉토리, `-`는 일반 파일, `l`은 바로가기와 같은 링크 파일을 나타낸다.
2. 첫 번째 `rwx`는 소유자의 권한을 나타낸다.
3. 두 번째 `rwx`는 소유 그룹에 포함된 유저들의 권한을 나타낸다.
4. `r-x`는 그 외 유저들의 권한을 나타낸다.

여기서 `r` = Read, `w` = Write, `x` = Execute 를 의미한다.

Owner, Group은 소유자와 소유 그룹을 나타낸다. 위 예시에서는 둘 다 `user`이다.

### 파일 및 디렉토리 권한 명령어

```bash
chmod <권한> <파일명>              # 파일 권한 변경 (root 또는 파일 소유자만 실행 가능)
chown 사용자명[.그룹명] 파일명      # 파일 소유자 혹은 소유 그룹 변경 (root만 실행 가능)
```

### 특수 권한

- **setuid** : 일반 사용자가 파일을 실행하면 파일 소유자 권한으로 실행된다.
- **setgid** : 일반 사용자가 파일을 실행하면 파일 소유 그룹의 권한으로 실행된다.
  - 둘 다 `x` 대신 `s` 문자로 나타낸다. 대문자 `S`로 표시된 경우는 실행 권한이 없는 경우이다.
- **sticky bit** : 파일 및 디렉토리 소유자와 root 사용자 외의 일반 사용자는 파일을 삭제할 수 없다.
  - `x` 대신 `t`로 나타내며, 마찬가지로 실행 권한이 없는 경우에는 대문자 `T`로 나타낸다.

## 디렉토리 구조

| 디렉터리 | 설명 |
| --- | --- |
| `/bin` | 일반 유저가 사용할 수 있는 기본적인 명령어나 프로그램을 담고 있는 디렉터리 |
| `/boot` | 시스템 부팅에 필요한 파일들을 담고 있는 디렉터리 |
| `/dev` | 컴퓨터에 부착된 물리적인 장치들을 파일 형태로 나타내는 파일들을 담고 있는 디렉터리 (리눅스는 디바이스 드라이버를 거쳐 파일처럼 접근 가능) |
| `/etc` | 운영체제나 그 위에서 동작하는 서비스의 설정 파일들을 담고 있는 디렉터리 |
| `/home` | 각 일반 유저의 홈 디렉터리를 담고 있는 디렉터리. 예를 들어 `dream` 유저의 홈 디렉터리는 `/home/dream` |
| `/lib` | 시스템에 필요한 라이브러리 파일들을 담고 있는 디렉터리. `/bin`이나 `/sbin`의 프로그램이 필요로 하는 동적 라이브러리 파일이 여기에 존재 |
| `/opt` | 소프트웨어 패키지들을 담는 디렉터리 |
| `/proc` | 리눅스 커널 자원에 접근할 수 있는 파일과 프로세스를 나타내는 파일을 담고 있는 디렉터리 |
| `/root` | `root` 유저의 홈 디렉터리 |
| `/sbin` | `/bin`과 마찬가지로 기본적인 명령어나 프로그램을 담고 있으나, `root` 유저가 사용할 수 있는 것들 |
| `/tmp` | 유저나 프로그램이 임시로 파일을 생성해야 할 때 사용하는 디렉터리. 오래 존재한 파일은 자동 삭제되므로 주의 필요 |
| `/usr` | 사용자 바이너리, 문서, 라이브러리, 헤더 파일 등을 담고 있는 디렉터리 |
| `/var` | 프로그램이나 시스템이 실시간으로 가변적인 파일을 사용하고 저장해야 할 때 활용하는 디렉터리. 예를 들어 `/var/log`에는 다양한 로그 파일이 저장됨 |
