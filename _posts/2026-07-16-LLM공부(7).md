Transformer가 실제로는 어떤 구조로 동작하고 있고 HuggingFace에서 제공하는 Transformer 라이브러리를 어떻게 사용하는지에 대한 정리이다.

# Transformer의 구조와 학습방식

## Transformer의 구조

용어정리 : Architecture = 학습 모델의 뼈대, Check Point = 학습된 가중치들

![이미지](/assets/img/2026-07-16-LLM공부(7)/트랜스포머.png)

### 인코더 + 디코더를 사용하는 구조(Seq2Seq Model)
Transformer 논문(Attention is All You Need)에서 발표된 원본 구조로, 번역처럼 입력을 이해해서 다른 형태의 출력을 생성하는 작업을 위해 만들어졌다. 인코더가 입력 문장을 이해하고, 디코더가 그 이해를 바탕으로 출력 문장을 생성한다.
- 대표 모델: T5, BART
- 대표 작업: 번역, 요약(생성형)

### 인코더만 사용하는 구조(Encoder Model)
입력 문장 전체를 양방향(앞뒤 동시에)으로 살펴봐서, 문맥이 반영된 벡터(은닉상태)로 변환하는 데 특화된 모델이다. 문장을 "이해"하는 데 강점이 있다.  
주어진 문장에 랜덤으로 마스킹을 진행하고 이를 재구성 하게끔 하는 방식으로 학습을 진행한다.  
전체 문장에 대한 이해를 요구하는 작업에 특화 되어있다.
- 대표 모델: BERT
- 대표 작업: 문장 분류, 개체명 인식, 질의응답

### 디코더만 사용하는 구조(Decoder Model)
지금까지 나온 단어들만 순차적으로(왼쪽→오른쪽) 참고해서 다음에 올 단어를 하나씩 예측한다. 앞에 있는 단어들에 대해서만 접근할 수 있다. 이러한 모델을 auto-regressive 라고 부른다.  
문장 내 다음 단어 예측을 반복하는 방식으로 이루어진다.
텍스트를 생성하는 작업에 특화 되어있다.
- 대표 모델: GPT
- 대표 작업: 텍스트 생성, 대화 생성

### Model Head
![이미지](/assets/img/2026-07-16-LLM공부(7)/ModelHead.png)  
Model Head는 hidden state의 고차원 벡터를 입력받아 다른 차원으로 투영한다.  
보통 하나 이상의 선형 레이어로 구성되며 Model head의 출력이 전체 Model의 출력이 된다.  
Task 종류에 따라 Head(분류Head, 토큰분류Head, QAHead)는 달라지며 Body부분의 가중치는 이미 학습된 가중치를 그대로 활용하지만 학습 과정에서 조금씩 미세조정된다.


## Transformer의 학습방식
**자가지도학습 (Self-Supervised Learning)** : 
정답(Label)을 사람이 만드는 게 아니라, 원본 데이터 안에서 스스로 만들어서 학습하는 방식.   
Label을 사람이 아닌 데이터 스스로 생성한다는 점에서 비지도학습과는 다르다.

![이미지](/assets/img/2026-07-16-LLM공부(7)/전이학습.png)

- **사전학습 (Pre-training)** : 
본 작업을 하기 전에 대량의 데이터로 기본적인 언어 지식을 미리 학습시켜 두는 것. (주로 자가지도학습 방식으로 진행)

- **미세조정 (Fine-tuning)** : 
사전학습된 모델을 가져와서, 내가 풀고 싶은 특정 작업에 맞게 추가로 학습시키는 것

- **전이학습 (Transfer Learning)** : 
한 작업에서 배운 지식을 다른 작업에 재활용하는 것. 사전학습 + 미세조정을 포괄하는 상위 개념.

# Hugging Face Transformers

**Hugging face** : AI 모델 허브와 라이브러리를 만드는 플랫폼

**Transformer lib** : 허깅페이스에서 지원하는 AI ML 모델을 가져올 수 있는 라이브러리

## Pipeline

Pipeline은 모델에 있어서 필수 과정인 전처리와 후처리 과정을 모델과 연결해주는 Transformer 라이브러리의 핵심 함수이다.

### 전체과정

![이미지](/assets/img/2026-07-16-LLM공부(7)/파이프라인동작방식.png)

1. 텍스트 전처리 과정을 거친다.
2. 전처리된 데이터를 모델 입력 넣는다.
3. 예측값이 후처리를 거쳐 사람이 이해할 수 있는 형태로 반환한다.

### Task와 Model
 ```python
from transformers import pipeline

example = pipeline("Task Name", model="Model Name(비워두면 기본모델)")
```

| 대표적인 Task 종류 | 설명 |
|---|---|
| feature-extraction | 특징 추출 (텍스트에 대한 벡터 표현 추출) |
| fill-mask | 마스크 채우기 |
| ner | 개체명 인식 (Named Entity Recognition) |
| question-answering | 질의 응답 |
| sentiment-analysis | 감정 분석 |
| summarization | 요약 |
| text-generation | 텍스트 생성 |
| translation | 번역 |
| zero-shot-classification | 제로샷 분류 |
| text-classification | 문장 분류 |
| token-classification | 토큰 분류 |

**Model 가져오기**  

![이미지](/assets/img/2026-07-16-LLM공부(7)/image.png)

    1. 파인튜닝까지 완료된 모델

    2. 사전 학습만 진행한 모델을 사용한다.

    중요한 부분은 사전학습때 사용한 토크나이저를 그대로 사용해야한다. (보통 세트로 같이 있다.)

    찾은 모델은 **체크포인트(모델 경로/이름)**로 가져와서 사용한다.

# Tokenizer + Model + 후처리 분리해서 설명
## Tokenizer 
NLP의 핵심요소, 원시텍스트를 숫자로 변환시켜 컴퓨터가 알아볼 수 있게 바꾸는 과정  
정확히는 텍스트 → 숫자(ID)로 바꾸는 것 까지만 담당한다. (Word Embedding은 모델이 담당)

### 단어기반 토큰화

1. 단어기반 토크나이저를 사용하면 아주 큰 “단어 사전” 을 얻게된다.
2. 각 단어에 대한 식별자가 필요하다.
3. OOV 토큰 처리가 필요하다.

### 문자기반 토큰화

1. 단어사전이 간결해진다.
2. OOV토큰의 수가 줄어든다.
3. 문자의 의미가 더 작아진다.
4. 처리해야하는 토큰의 수가 늘어난다.

### 서브워드 토큰화

1. 단어기반의 장점 + 문자기반의 장점
2. 자주 사용되는 단어는 그대로 사용하고, 희귀하거나 자주 사용되지 않는 단어는 의미를 가진 서브워드로 나눈다.
3. “Let’s do tokenization!”을 토큰화 한다면  
![alt text](/assets/img/2026-07-16-LLM공부(7)/토큰화예시.png)

### 이외의 다양한 토큰나이저
GPT-2 : Byte-level BPE  
BERT : WOrdPiece  
또한 SetencePiece와 Unigram등이 있다.  

### 예시코드 : 토크나이저 가져오기

```python
from transformers import BertTokenizer, AutoTokenizer

tokenizer = BertTokenizer.from_pretrained("bert-base-cased")

tokenizer = AutoTokenizer.from_pretrained("bert-base-cased")

tokenizer("Using a Transformer network is simple")
```
AutoTokenizer는 checkpoint("bert-base-cased")에 있는 토크나이저를 자동으로 가져와서 반영해준다.

### 예시코드 - 다중시퀀스 처리
```python
import torch
from transformers import AutoTokenizer, AutoModelForSequenceClassification

checkpoint = "distilbert-base-uncased-finetuned-sst-2-english"
tokenizer = AutoTokenizer.from_pretrained(checkpoint)
model = AutoModelForSequenceClassification.from_pretrained(checkpoint)
pad = tokenizer.pad_token_id

# 길이가 다른 두 시퀀스
seq1 = [[200, 200, 200]]
seq2 = [[200, 200]]

# 패딩만 해서 배치로 묶음
batched = [[200, 200, 200],
           [200, 200, pad]]
mask    = [[1, 1, 1],
           [1, 1, 0]]        # 0 = 패딩이니 무시하라

# 각각 출력 값 (정답)
print(model(torch.tensor(seq1)).logits)     # [[ 1.5694, -1.3895]]
print(model(torch.tensor(seq2)).logits)     # [[ 0.5803, -0.4125]]

# 마스크 X -> seq2가 틀림
print(model(torch.tensor(batched)).logits)
# [[ 1.5694, -1.3895],
#  [ 1.3373, -1.2163]]  ✗

# 마스크 O → seq2와 일치
print(model(torch.tensor(batched), attention_mask=torch.tensor(mask)).logits)
# [[ 1.5694, -1.3895],
#  [ 0.5803, -0.4125]]  ✓
```
### 예시코드 - 토크나이저를 이용한 처리

```python
import torch
from transformers import AutoTokenizer, AutoModelForSequenceClassification

checkpoint = "distilbert-base-uncased-finetuned-sst-2-english"
tokenizer = AutoTokenizer.from_pretrained(checkpoint)
model = AutoModelForSequenceClassification.from_pretrained(checkpoint)

sequences = ["I've been waiting for this my whole life.", "So have I!"]

tokens = tokenizer(sequences, padding=True, truncation=True, return_tensors="pt")

output = model(**tokens)

print(output.logits)
# [[-1.5607,  1.6123],
#  [-3.6183,  3.9137]]
```

## Model
```py
from transformers import AutoModelForSequenceClassification

checkpoint = "distilbert-base-uncased-finetuned-sst-2-english" #체크포인트 가져오기
model = AutoModelForSequenceClassification.from_pretrained(checkpoint) #아키텍쳐에 체크포인트값을 반영해서 model 생성
```
AutoModel은 checkpoint에 있는 모델을 자동으로 반영해준다.

AutoModelFor...이 나오면 특정 Task에 대한 체크포인트값을 반영해준다. (AutoModelForTranslation라면 번역 Task에 대한 가중치들을 업데이트 해준다.)

**from_pretrained()** = 저장된 모델을 로딩하라는 메소드  
**save_pretrained()** = 현재 모델을 저장하라는 메소드
```python
#저장 예제 코드
from transformers import AutoTokenizer, AutoModelForSequenceClassification

checkpoint = "distilbert-base-uncased-finetuned-sst-2-english"

tokenizer = AutoTokenizer.from_pretrained(checkpoint)
model = AutoModelForSequenceClassification.from_pretrained(checkpoint)

# 모델과 토크나이저를 로컬 디렉토리에 저장
model.save_pretrained("path입력")
tokenizer.save_pretrained("path입력")
```
directory_on_my_computer/  
1. config.json -> 모델 아키텍쳐 : 모델저장
2. pytorch_model.bin -> 모델 가중치 값 : 모델저장  
3. tokenizer_config.json -> 토크나이저 설정 : 토크나이저 저장
4. vocab.txt -> 어휘 사전 : 토크나이저 저장
5. special_tokens_map.json -> 특수 토큰 정보 : 토크나이저 저장  
등등 있다.

## 후처리
모델에서의 출력값은 다음과 같다.
1. AutoModel를 사용했을 때 출력 값 : Hidden state
2. AutoModelFor...(TaskHead까지 포함된 경우) : 가공되지 않은 Head의 출력 값 ex) 감정분류모델 : tensor([[-1.5607, 1.6123]])

어떻게 해야지 사람이 보기 쉬운 형태로 받을 수 있는지
    
```py
    import torch
    from transformers import AutoTokenizer, AutoModelForSequenceClassification

    checkpoint = "distilbert-base-uncased-finetuned-sst-2-english"
    tokenizer = AutoTokenizer.from_pretrained(checkpoint)
    model = AutoModelForSequenceClassification.from_pretrained(checkpoint)

    inputs = tokenizer("I've been waiting for this my whole life.", return_tensors="pt")
    with torch.no_grad():
        logits = model(**inputs).logits      # tensor([[-1.5607, 1.6123]])

    probs = torch.softmax(logits, dim=-1)    # tensor([[0.0405, 0.9595]])
    idx = probs.argmax(dim=-1).item()        # 1
    print(model.config.id2label[idx], probs[0][idx].item())
    # POSITIVE 0.9595
```
    
`probs = torch.softmax(logits, dim=-1)    # tensor([[0.0405, 0.9595]])`가 핵심으로 softmax를 사용해 확률 형태로 변환한다.   
id2label는 추론결과를 해석할 때 사용된다. 기계 -> 사람 번역기로 생각하면 편하다.
