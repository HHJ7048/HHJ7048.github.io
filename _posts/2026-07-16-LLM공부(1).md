## 1. 토큰화(Tokenization)

텍스트 처리에서 가장 먼저 진행되는 단계이다. 기계가 데이터를 읽을 수 있게 토큰 단위로 나누는 방식이다. 여기서 토큰이란 단어, 문장, 형태소 등등 다양한 종류가 있다.

### 기본 용어 설명

- 주어진 텍스트 → Corpus
- 원하는 단위 → Token
- 전체 과정 → Tokenization

## 2. 정제, 정규화, 어간 추출, 표제어 추출, 불용어 제거

### 정제(Cleaning), 정규화(Normalization)

코퍼스에서 노이즈 데이터를 제거하거나, 표현 방법이 다른 단어들을 통합시켜서 같은 단어로 만들어준다.
데이터의 크기를 줄여주거나 학습 정확도를 높여준다.

1. 규칙 기반 통합

    ex) USA -> US

2. 대, 소문자 통합

    ex) ApPle = apple = APPLE

3. 불필요한 단어 제거

    불용어(Stopword)나 길이가 너무 짧은 단어, 등장 빈도가 너무 작은 단어 등을 제거한다.

4. 어간 추출(Stemming)

    단어의 의미를 담고있는 핵심 부분을 추출

5. 표제어 추출(Lemmatization)

    기본 사전 단어로 변환 시키는 방식

## 3. 정수 인코딩

### 1. Integer Encoding

단어를 빈도수에 대한 집합으로 만든 뒤, 빈도수가 높은 순서부터 정수를 차례대로 부여한다. 그리고 기존 문장을 인코딩된 숫자로 표현할 수 있다.

![이미지](/assets/img/2026-07-16-LLM공부(1)/image(8).png){: width="600" }

![이미지](/assets/img/2026-07-16-LLM공부(1)/image(9).png){: width="600" }

정수의 크기가 학습간에 잘못된 정보를 만들어낼 수 있다.

ex) '딸기' = 1, '바나나' = 2 라고 Encoding 되어있다면 딸기 + 딸기 = 바나나 와 같은 잘못된 수식을 만들어 낼 수 도 있다.

이를 방지하기 위해 인코딩된 정수를 Vectorization 시켜서 방지한다.

\+ 물론 순서나 크기의 정보가 도움이 되는 단어도 있을 수 있다.

ex) 1층, 2층, 3층 / 10대, 20대, 30대 등등
